⚠️ Risici & mitigering
Kan ikke finde kandidat inden 7. april
Rekruttering tager tid. Hvis ansættelse trækker ud, forsvinder ramp-up-buffer.
→ Brug netværk + freelance-platforme parallelt. Overvej contract-to-hire. Selv med start medio april er der stadig 28+ tickets kapacitet.
Ramp-up tager længere end 3 uger
Kompleks codebase, mange domæner. Ny dev kan have brug for mere tid.
→ Start med FMS (isoleret, lav risiko). Hold scope snævert de første 2 sprints. Selv med 4 ugers ramp-up er der +24 tickets kapacitet.
Ny dev sidder fast på Michaels off-dage (man+ons)
Backend-spørgsmål kan kræve Michael-viden som Jakob/Tony ikke har.
→ Michael forbereder skriftlig onboarding-doc (arkitektur, API-map, domæne-glossar) inden dag 1. Ny dev samler spørgsmål til ti+to. Jakob kan svare produkt-spørgsmål, Tony kan svare infra-spørgsmål.
Dårligt kultur-/team-fit
Lille team — én person kan påvirke dynamikken negativt.
→ Prøveperiode med klare milestones (uge 1: første PR, uge 3: selvstændige tickets). Evaluering efter 2 uger.
